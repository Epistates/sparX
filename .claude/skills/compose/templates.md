# Post Templates — Proven Structures by Format

## Hook Templates

### Curiosity Gap
```
[Surprising result] — and it only took [minimal effort/time].
```
Example: "Cut our API latency by 73% — and it only took changing one line."

### Contrarian Take
```
Unpopular opinion: [common belief] is wrong because [specific reason].
```
Example: "Unpopular opinion: writing more tests is making your code worse."

### Specific Number
```
[Exact number] [result] in [timeframe]. Here's how:
```
Example: "847 new GitHub stars in 2 weeks. Here's exactly what changed."

### Pain Point
```
[Audience] are still [doing frustrating thing]. There's a better way:
```
Example: "Most devs are still writing Dockerfiles by hand. There's a better way:"

### Before/After
```
[Before state] → [After state]. What changed:
```
Example: "90-second builds → 8-second builds. What changed:"

### Question Hook
```
Why does [common thing] still [frustrating behavior] in [current year]?
```
Example: "Why does npm install still take 45 seconds in 2026?"

### Social Proof Lead
```
[Impressive metric] and counting. [What it is]:
```
Example: "10,000 deploys and zero rollbacks. Here's our setup:"

## Single Post Templates

### The Insight Post
```
[Hook — surprising observation or result]

[2-3 lines explaining why this matters]

[Specific proof point or example]

[Question inviting replies]
```

### The Announcement Post
```
[What's new — specific feature/version with a bold claim]

[Why it matters to the reader (not to you)]

[Proof — number, benchmark, comparison]

[CTA — question or "try it" without a direct link]
```

### The Hot Take
```
[Strong opinion stated directly]

[Brief supporting reasoning — 2-3 sentences max]

[Rhetorical question or "Change my mind"]
```

### The Quick Tip
```
[Platform/tool] tip that saves [specific time/effort]:

[Concise how-to — 2-3 lines]

[Before/after or "I wish I knew this sooner"]
```

### The Build-in-Public Update
```
[What just happened — specific milestone]

[Why it was harder/easier/different than expected]

[Screenshot or metric]

[What's next + question]
```

### The Benchmark/Comparison
```
[Tool A] vs [Tool B] — tested on [specific workload]:

[Result with specific numbers]
[Screenshot/table if applicable]

Surprised? What are you running?
```

## Reply Templates (for link follow-ups)

### Link Drop Reply
```
Link: [URL]

[One line of additional context that makes clicking worth it]
```

### Resource Reply
```
Resources mentioned:
→ [Resource 1]: [URL]
→ [Resource 2]: [URL]

[Brief qualifier — "all free" / "the first one is the most important"]
```

## Poll Templates

### Binary Opinion
```
[Provocative framing of the choice]

Which side are you on?

🔵 [Option A]
🔴 [Option B]

Reply with your reasoning ↓
```

### Tool/Approach Comparison
```
For [specific use case], which do you reach for?

[Option 1]
[Option 2]
[Option 3]
[Other — reply below]
```
