# Content Formats — Format-Specific Optimization

## Single Post (Standard Tweet)

**Best for**: Quick insights, hot takes, questions, announcements, visual proof

### Anatomy of a High-Scoring Single Post

```
[HOOK — first 8-12 words that stop the scroll]

[BODY — value delivery, 2-4 lines max for single posts]

[PROOF — number, screenshot, specific result]

[CTA — question or soft ask that triggers replies]
```

### Character Length Sweet Spots
- **Optimal**: 200-280 characters (enough substance for dwell, not overwhelming)
- **Short punchy**: 80-140 characters (for strong opinions/observations)
- **Long form**: 500-1000 characters (for insight-dense posts — use line breaks)
- **Avoid**: 140-200 character range (awkward middle — neither punchy nor substantive)

### Formatting for Readability
- One thought per line
- Blank lines between sections
- Short sentences (12-18 words)
- Sentence fragments are fine on X
- No walls of text — create visual breathing room

## Threads

**Best for**: Deep dives, tutorials, build-in-public, storytelling, benchmark breakdowns

### Optimal Thread Structure (7 tweets sweet spot)

| Position | Purpose | Template |
|----------|---------|----------|
| Tweet 1 | **Hook** | Promise specific value: "How I [achieved X] in [timeframe] (thread)" |
| Tweet 2 | **Context/Problem** | Set up the pain point or situation |
| Tweet 3-5 | **Value Delivery** | One insight per tweet, each standalone readable |
| Tweet 6 | **Proof/Results** | Numbers, screenshots, before/after |
| Tweet 7 | **CTA** | Question + soft follow ask |

### Thread Rules
- Each tweet must stand alone (people may see individual tweets in feed)
- Don't number tweets "1/", "2/" etc. — the thread indicator handles this
- First tweet is the most important — it determines whether anyone reads the rest
- Include an image/screenshot in at least one middle tweet (dwell boost)
- End thread tweets with a hook to the next ("But then I found something...")
- Threads get 3× engagement vs equivalent single posts
- The algorithm treats threads as one content unit for the diversity filter

### Thread Length Guidelines
- **5-7 tweets**: Optimal engagement zone
- **3-4 tweets**: Fine for quick tutorials or listicles
- **8-10 tweets**: Use for deep technical breakdowns — risk of drop-off after tweet 7
- **10+ tweets**: Only for exceptional content — most readers drop off

## Polls

**Best for**: Community engagement, opinion gathering, niche positioning

### Poll Optimization
- 2-4 options (2 creates stronger "pick a side" dynamic)
- Duration: 24 hours (urgency) or 7 days (steady engagement)
- Frame as a genuine question, not a leading one
- Add context in the tweet text above the poll
- Controversial but not toxic — "Which is better: X or Y?" drives votes + quote tweets

### Example Structure
```
[Provocative question or observation]

[Brief context — why this matters]

🔵 Option A
🔴 Option B

[Optional: "Reply with why"]
```

## Quote Tweets

**Best for**: Adding commentary, starting debates, amplifying others, building relationships

### Quote Tweet Strategy
- Add genuine insight, not just agreement
- Your commentary should provide new value beyond the original
- Both you and the original author benefit from the engagement
- Use when your take adds a different angle
- Quote tweets of viral posts get partial distribution benefit from the original

## Image/Screenshot Posts

**Best for**: Benchmarks, code snippets, data, before/after comparisons, proof

### Image Optimization
- Single image gets 1.5-2× engagement vs text-only
- Multi-image (2-4) carousel format works well for comparisons
- Text overlay on images increases dwell (people read the image)
- Screenshots of terminal output, code, or data tables create high dwell
- Alt text is indexed for search — include relevant keywords

### Dimension Guidelines
- Single image: 1200×675px (16:9) or 1080×1080px (1:1)
- Pair: 700×800px each
- Four-grid: 600×335px each

## Video Posts

**Best for**: Demos, tutorials, before/after, screen recordings

### Video Optimization
- **15-45 seconds** is the sweet spot (high completion rate)
- First 3 seconds must hook — no intros, no logos, start with the action
- Captions/subtitles are essential (most video is watched muted)
- Screen recordings of actual product/code in action perform exceptionally
- Completion rate is the key metric — shorter videos complete at higher rates
- Remember: text outperforms video by ~30% on X — video is not always the answer
- Best strategy: **text + video combo** (text for the algorithm, video for dwell time)

### Video Length Guidelines
| Length | Best For | Completion Rate |
|--------|----------|----------------|
| 5-15s | Quick demos, GIFs, one-feature showcase | Very high |
| 15-45s | Tutorials, benchmarks, product demos | High |
| 45-90s | Deep tutorials, multi-step demos | Moderate |
| 90s+ | Only for compelling content with strong narrative | Low (risky) |

## Build-in-Public Updates

**Best for**: Showing progress, building audience connection, creating narrative arcs

### Structure
```
[What happened — specific achievement or milestone]

[Why it matters — context that makes non-experts care]

[What's next — creates anticipation for follow-up]

[Visual proof — screenshot, metric, or demo]
```

### Cadence
- 2-3 per week maximum (more feels like spam)
- Space out with other content types
- Create narrative arcs: problem → attempt → failure → breakthrough → result
- Reference previous updates to reward returning readers
